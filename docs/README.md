## Learning Path

### Day 1: Understanding (2-3 hours)
1. Read [BEGINNER_GUIDE.md](./BEGINNER_GUIDE.md) completely
2. Refer to [GLOSSARY.md](./GLOSSARY.md) for any unclear terms
3. Draw the architecture on paper
4. Explain it to yourself out loud

### Day 2: Building (3-4 hours)
1. Follow the "How to Run Everything" section
2. Run Producer locally
3. Build and deploy Consumer
4. Test end-to-end
5. Keep [QUICK_REFERENCE.md](./QUICK_REFERENCE.md) open for commands

### Day 3: Troubleshooting (1-2 hours)
1. Break something on purpose
2. Practice fixing it using troubleshooting guides
3. Run through the complete flow again
4. Make sure you understand each step

### Day 4: Presentation Prep (3-4 hours)
1. Read [PRESENTATION_TIPS.md](./PRESENTATION_TIPS.md)
2. Practice the demo 3-5 times
3. Prepare slides or diagrams
4. Practice answering Q&A
5. Record yourself presenting

### Day 5: Final Review (1 hour)
1. Run through demo one more time
2. Review Q&A answers
3. Check all resources are working
4. Get good sleep!

**Total time:** 10-14 hours spread over 5 days
